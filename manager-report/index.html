§01

Executive summary

⚠

Business problem

Strategy Builder backtests hit HTTP gateway timeouts. Users see “Oops timeout error” after 60+ seconds — blocking strategy validation and deployment.

⚡

Technical solution

ODIN intercepts backtest requests via POST /v1/backtest/strykex. OHLC served from a 4-tier read path; indicators precomputed nightly; conditions evaluated in a Numba JIT kernel.

✓

Delivery ask

Approve StrykeX API redirect (USE_ODIN=true), provision Redis + ETL cron, and provide production QuestDB credentials. No frontend work required.

Plain English: We built a speed layer behind StrykeX. Traders use the same UI; only the backtest backend gets faster, cached, and reliable.
§02

Problem statement

When a user completes the StrykeX builder flow (NIFTY → 5 Minute → Entry/Exit rules → Backtest), the synchronous backend path fails under load:

Root causeTechnical detailUser impact
Unbounded OHLC fetchFull 5-year QuestDB scan per request; no partition pruning at API layer60s+ wait
Redundant indicator computeRSI, EMA, MACD recomputed in Pandas on every candle, every clickCPU-bound
Single-threaded evaluationPython/Pandas condition loop over ~20k bars/yearSlow
Gateway timeoutLoad balancer cuts connection before JSON responseError screen
ODIN positioning: Drop-in backend service between StrykeX API and QuestDB. Strategy Builder UI remains unchanged.

§05

Multi-tier data plane

Read path for OHLC + indicators (ordered by latency):

L0HotStore (RAM)In-process preloaded slices — 90d & 365d NIFTY 5m. Sub-ms reads. Populated at API startup via prewarm_defaults().
L1Redis 7Arrow IPC blobs, 24h TTL. Keys: ohlc:NIFTY:5m:{start}:{end}, indicators:NIFTY:5m:{start}:{end}
L2Parquet (SSD)Monthly columnar files via pl.scan_parquet. 10–30× faster than SQL range scans. Partitioned by YYYY-MM.
L3QuestDBOHLC source of truth. Bounded HTTP SQL only — symbol, timeframe, timestamp range in WHERE clause.
PGPostgreSQLIndicator catalog (96 slugs), indicator_bars, bar_extensions JSONB. Not in OHLC read path.

§08

Indicator engine

Tier A — Nightly precompute

EMA 9/20/50/200, SMA 20/50, RSI 14, MACD, ATR 14, Bollinger 20/2 → stored in Parquet via precompute_all()

Tier B — On-demand + cache

Custom params (e.g. RSI 21) computed once with Polars, cached in Redis: ind:NIFTY:5m:RSI:{hash}:{start}:{end}

Catalog

96 indicators + 286 rule templates in PostgreSQL. Exposed via GET /v1/indicators and GET /v1/indicators/{slug}/rules

Execution kernel: Polars prepares aligned NumPy arrays → @njit run_backtest_kernel() evaluates multi-rule AND conditions per candle → Python builds trade list + equity curve.

§10

StrykeX integration

Two backend hooks. Zero frontend changes.

  1. Feature flag: USE_ODIN=true + ODIN_API_URL=http://odin-api:8000 in StrykeX backend
  2. Prefetch on Strategy Setup: when user confirms NIFTY + 5m, fire POST /v1/prefetch (async, fire-and-forget)
  3. Backtest redirect: replace inline backtest with POST /v1/backtest/strykex

Complete

OHLC + indicator slice keys; 24h TTL; /metrics cache hit rate.
