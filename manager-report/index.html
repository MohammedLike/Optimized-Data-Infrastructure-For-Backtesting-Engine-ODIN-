😤 The problem users face today

When a trader builds a strategy on StrykeX and clicks Backtest, they often get an error — the system takes too long and gives up.

Why it breaks

  • Too much data — the system pulls years of price history every single click
  • Repeats the same work — indicators like RSI are calculated from scratch every time
  • Slow processing — one request does everything in a single slow path
  • User gives up — after ~60 seconds, the page shows a timeout

✅ The fix — Project ODIN

ODIN (Optimized Data Infrastructure) sits behind StrykeX. Users see the same website. Only the back-end gets smarter and faster.

No UI changes. Traders keep using Strategy Builder exactly as today. We only change where the backtest request goes.

Think of it like a restaurant

❌ Today Every order sends the waiter to the farm, harvests crops, cooks from scratch — then the customer waits an hour.
→
✅ With ODIN Popular dishes are prepped overnight. The kitchen grabs ready ingredients. Orders come out in minutes.

What ODIN does (plain English)

PieceWhat it does

Memory cacheRemembers recent requests (Redis) — helps repeat clicks
RAM pre-loadData loaded into server memory at startup — makes the first click fast too

Fast engineRuns the strategy rules on optimized code instead of slow Python loops

🔌 StrykeX integration — for the dev team

Two small backend changes. No new screens for traders.

Change 1 — Send backtests to ODIN

Instead of running the backtest inline, StrykeX API calls ODIN:

POST http://localhost:47100/v1/backtest/strykex Content-Type: application/json { strategy fields from Strategy Builder }

Set USE_ODIN=true and ODIN_API_URL in StrykeX backend config.

Change 2 — Prefetch when user picks NIFTY + 5m

Call this as soon as the user selects instrument and timeframe (steps 1–2 of the builder). By the time they reach Entry/Exit and click Backtest, data is in memory.

POST http://localhost:47100/v1/prefetch Content-Type: application/json { "symbol": "NIFTY", "timeframe": "5m", "days": 365 }

Suggested trigger: on Strategy Setup screen when NIFTY + 5 Minute is confirmed. Fire-and-forget — ignore errors silently.

Ops checklist (after nightly data update)

  1. Export new prices: python -m services.nightly_etl.export
  2. Pre-calculate indicators: python -m services.nightly_etl.precompute
  3. Refresh RAM cache: python scripts/prewarm_cache.py

Steps 1–3 also run automatically when ODIN API restarts if PREWARM_ON_STARTUP=true.

For managers: Dev work is ~2 API hooks + a nightly script. No product redesign.
